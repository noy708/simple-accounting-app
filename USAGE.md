# 使用方法ガイド

## Simple Accounting App の基本的な使い方

### 1. アプリケーションへのアクセス

1. Webブラウザで `http://localhost:3000` にアクセス
2. アプリケーションのメイン画面が表示されます

### 2. 画面構成

アプリケーションは3つの主要なセクションで構成されています：

#### 2.1 残高表示エリア（上部）
- 現在の残高が大きく表示されます
- プラスの場合は緑色、マイナスの場合は赤色で表示
- 取引を追加すると自動的に更新されます

#### 2.2 取引入力フォーム（中央）
- 新しい取引を入力するためのフォームです
- 以下の項目を入力できます：
  - **金額**: 取引の金額（必須）
  - **説明**: 取引の内容説明（必須）
  - **種類**: 収入または支出を選択（必須）
  - **日付**: 取引日（デフォルトは今日）

#### 2.3 取引履歴（下部）
- 過去の取引が一覧表示されます
- 新しい取引が上に表示されます（日付順）
- 収入は緑色、支出は赤色のチップで表示

### 3. 取引の入力方法

#### 3.1 収入の入力例

1. **金額**欄に `50000` と入力
2. **説明**欄に `給与` と入力
3. **種類**で `収入` を選択
4. **日付**を確認（必要に応じて変更）
5. **追加**ボタンをクリック

#### 3.2 支出の入力例

1. **金額**欄に `1200` と入力
2. **説明**欄に `昼食代` と入力
3. **種類**で `支出` を選択
4. **日付**を確認（必要に応じて変更）
5. **追加**ボタンをクリック

### 4. 入力規則とバリデーション

#### 4.1 必須項目
- すべての項目が必須です
- 空欄がある場合はエラーメッセージが表示されます

#### 4.2 金額の入力規則
- 正の数値のみ入力可能
- 小数点以下2桁まで対応
- 0以下の値は入力できません

#### 4.3 説明の入力規則
- 最大200文字まで入力可能
- 空文字は入力できません

#### 4.4 日付の入力規則
- 有効な日付形式で入力
- 未来の日付も入力可能

### 5. 取引履歴の見方

#### 5.1 表示項目
- **日付**: 取引が行われた日付
- **説明**: 取引の内容
- **金額**: 取引金額
- **種類**: 収入（緑）または支出（赤）のチップ

#### 5.2 並び順
- 最新の取引が上に表示されます
- 同じ日付の場合は、作成時刻順で表示

### 6. 残高の計算

- 残高 = 全収入の合計 - 全支出の合計
- 取引を追加するたびに自動的に再計算されます
- プラスの場合は緑色、マイナスの場合は赤色で表示

### 7. エラーメッセージ

#### 7.1 入力エラー
- **「金額は必須です」**: 金額が入力されていない
- **「金額は正の数値である必要があります」**: 0以下の値が入力された
- **「説明は必須です」**: 説明が入力されていない
- **「日付は必須です」**: 日付が選択されていない

#### 7.2 システムエラー
- **「取引の追加に失敗しました」**: サーバーエラーが発生
- **「データの読み込みに失敗しました」**: 通信エラーが発生

### 8. 使用例とシナリオ

#### 8.1 日常的な家計管理

**朝の収入記録:**
1. 金額: `250000`
2. 説明: `月給`
3. 種類: `収入`
4. 日付: `2025-01-01`

**昼の支出記録:**
1. 金額: `800`
2. 説明: `コンビニ弁当`
3. 種類: `支出`
4. 日付: `2025-01-01`

**夕方の支出記録:**
1. 金額: `3500`
2. 説明: `スーパーで食材購入`
3. 種類: `支出`
4. 日付: `2025-01-01`

#### 8.2 小規模事業の売上管理

**売上の記録:**
1. 金額: `15000`
2. 説明: `商品販売`
3. 種類: `収入`
4. 日付: `2025-01-01`

**経費の記録:**
1. 金額: `2000`
2. 説明: `事務用品購入`
3. 種類: `支出`
4. 日付: `2025-01-01`

### 9. ベストプラクティス

#### 9.1 説明の書き方
- 具体的で分かりやすい説明を心がける
- 例: ❌「買い物」→ ⭕「スーパーで食材購入」
- 例: ❌「収入」→ ⭕「フリーランス報酬」

#### 9.2 定期的な確認
- 毎日の終わりに残高を確認
- 週単位で支出パターンを確認
- 月単位で収支バランスを確認

#### 9.3 データの整理
- 似たような取引は説明を統一する
- 定期的な取引（家賃、光熱費等）は同じ説明を使用

### 10. トラブルシューティング

#### 10.1 画面が表示されない
1. ブラウザを更新してみる
2. 別のブラウザで試してみる
3. バックエンドサーバーが起動しているか確認

#### 10.2 取引が追加されない
1. すべての必須項目が入力されているか確認
2. 金額が正の数値になっているか確認
3. ネットワーク接続を確認

#### 10.3 残高が正しく表示されない
1. ページを更新してみる
2. ブラウザのキャッシュをクリア
3. 取引履歴を確認して手動で計算

### 11. キーボードショートカット

- **Tab**: 次の入力項目に移動
- **Shift + Tab**: 前の入力項目に移動
- **Enter**: フォーカスがボタンにある時に実行

### 12. モバイル対応

- スマートフォンやタブレットでも利用可能
- タッチ操作に最適化されたインターフェース
- 画面サイズに応じて自動的にレイアウトが調整

このガイドに従って、Simple Accounting Appを効果的に活用してください。
